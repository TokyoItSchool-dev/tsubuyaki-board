# 投稿者名フィールド拡張・頭文字アバター TDD計画

## 概要

投稿作成時に投稿者名を必須入力のまま扱い、任意のアバター背景色を選択できるようにする。

選択した色は投稿データとして DB に保存し、一覧画面・詳細画面で「頭文字 + 背景色」の正円アバターを投稿者名の左に表示する。

既存では投稿者名必須は実装済みのため、今回の中心は `avatarColor` の追加、フォーム入力、保存、表示、CSS 整備とする。

## インクリメント分割

1. Backend: 投稿にアバター色を保存できる
2. Form/Controller: 投稿フォームで色を選択して登録できる
3. View: 一覧画面で頭文字アバターを表示する
4. View: 詳細画面でも同じ頭文字アバターを表示する
5. Refactor: 頭文字・色の扱いを整理する

## 最初のインクリメント

### Backend: 投稿にアバター色を保存できる

#### 要件

- `posts` テーブルに `avatar_color` を保持できる。
- `Post` エンティティから `avatarColor` を取得できる。
- 新規投稿作成時、投稿者名・本文・作成日時に加えてアバター色を保存できる。
- 既存の投稿一覧・検索・詳細・いいね機能の動作は変えない。
- 未指定や空文字の色は `#3498db` を保存する。

## 前提

- アバター色は投稿ごとの属性として保存する。
- ユーザーアカウント概念は追加しない。
- 色の候補は HTML の color input による簡易選択とする。
- 初期デフォルト色は `#3498db`。
- 頭文字は投稿者名の先頭 1 文字を大文字化する。
