# いいねボタン レイアウト要件とTDD計画

## 概要

次のタスクでは、投稿詳細画面に表示する「いいねボタン」を実装する。

このドキュメントでは、まだ実装コードは書かず、レイアウト要件、インクリメント分割、
最初の Red TODO を定義する。

## レイアウト要件

### 配置

- いいねボタンは投稿詳細ページの投稿カード内に表示する。
- 表示位置は本文の下、作成日の上とする。
- 投稿カード内の表示順は以下とする。
  1. 投稿者名
  2. 本文
  3. いいねボタン
  4. 作成日
- いいねボタンは投稿カード内に収め、別カードや説明用の領域は追加しない。
- モバイル幅でも本文、いいねボタン、作成日が重ならないようにする。

### ボタン形状

- 完全なカプセル型にする。
- `border-radius: 9999px;` 相当の見た目にする。
- 背景色は白にする。
- 枠線は非常に薄いグレーの細線にする。
- 枠線の目安は `1px solid #e0e0e0`。
- 上下の余白は少し狭く、左右の余白はやや広めにする。
- ボタン単体で押せる要素だと分かる余白と枠線を持たせる。

### ボタン内の並び

ボタン内には、左から右へ以下を横一列に並べる。

1. ハートアイコン
2. いいね数
3. `いいね` テキスト

要件:

- ボタン内の要素は横並びにする。
- 上下中央が揃って見えるようにする。
- `display: flex; align-items: center;` 相当の配置にする。
- 要素間の余白は `gap` で管理する。
- いいね数は太文字にする。
- `いいね` テキストは通常の太さにする。
- ハートアイコンはピンクから赤色系にする。
- ハートは絵文字または既存導入済みのアイコンライブラリを使う。
- 新しいアイコンライブラリは、このボタンだけのためには追加しない。

### 表示文言

- いいね数が `1312` の場合、ボタン内は視覚的に `♡ 1312 いいね` のように見える。
- ハートの実体は絵文字、文字、またはアイコンでよい。
- 数値は model の `likeCount` を使う。
- 固定値をテンプレートに埋め込まない。

## インクリメント分割

1. 詳細画面の model に `likeCount` を積む。
2. 詳細画面で、本文の下かつ作成日の上にいいねボタンの表示枠を追加する。
3. いいねボタン内にハート、いいね数、`いいね` テキストを横並びで表示する。
4. いいねボタンをカプセル型にスタイリングする。
5. いいねボタンのフォーム送信先を `POST /posts/{id}/likes` にする。
6. CSRF トークンを含むフォームとして送信できるようにする。
7. モバイル幅でも崩れないことを Controller テストの HTML 構造と手動確認で検証する。

## 最初のステップ: 詳細画面にいいね数を渡す

最初のインクリメントは、見た目を作る前に、詳細画面で使う `likeCount` を
Controller から model に渡せる状態にすることに限定する。

このステップでは、CSS や Like ボタンの見た目はまだ実装しない。

## Red 要件

- `GET /posts/{id}` で投稿が存在する場合、詳細画面の model に `likeCount` を積む。
- `likeCount` の値は `PostService#countLikes(Long postId)` から取得する。
- 投稿が存在しない場合は、従来どおり 404 を返す。
- 投稿が存在しない場合、いいね数取得は呼ばない。

## Red TODO

`PostControllerTest` に以下の失敗テストを 1 本追加する。

**Controller: 詳細表示の model にいいね数を積む**

テスト名: `投稿詳細_存在するid_likeCountをmodelに渡す`

| 観点 | 内容 |
|---|---|
| Given | `postService.findById(1L)` が投稿を返す |
| Given | `postService.countLikes(1L)` が `1312` を返す |
| HTTP | `GET /posts/1` が 200 を返す |
| View | `posts/detail` |
| Model | `likeCount` に `1312` を渡す |

このテストが Red になる理由:

- `PostService#countLikes(Long postId)` がまだ存在しない。
- `PostController#detail` が `likeCount` を model に積んでいない。

## 後続 Red TODO

最初の Red を Green にした後、以下を 1 本ずつ追加する。

| 観点 | 内容 |
|---|---|

| HTML | いいね数用の要素がある |
| HTML | `いいね` テキスト用の要素がある |

**CSS: いいねボタンをカプセル型にする**

テスト名: `投稿詳細_いいねボタン_カプセル型のクラスを持つ`

| 観点 | 内容 |
|---|---|

|---|---|

## テスト計画

最初は対象 Controller テスト 1 本だけを実行し、Red を確認する。

```bash
./mvnw -B -Ph2 -Dtest=PostControllerTest#投稿詳細_存在するid_likeCountをmodelに渡す test
```

Green 後、詳細画面まわりの Controller テストを実行する。

```bash
./mvnw -B -Ph2 -Dtest=PostControllerTest test
```

ユースケース完了時に H2 プロファイルで verify を実行する。

```bash
./mvnw -B -Ph2 verify
```

## 前提

- このドキュメント作成では、実装コードやテストコードは変更しない。
- いいねボタンは投稿詳細画面にだけ表示する。
- 一覧画面へのいいねボタン追加は、このタスクには含めない。
- 新しいフロントエンドライブラリは追加しない。
- ユーザー入力由来の値を表示する場合は `th:text` を使い、`th:utext` は使わない。
