# 使ったプロンプトと評価

演習中に Codex CLI に投げた主要なプロンプトを記録する。
成果物に直結したプロンプトを 3 件記載する。

---

## プロンプト 1

**フェーズ**: 投稿基本機能

**プロンプト本文**:

```text
GET /posts の投稿一覧、/posts/new の投稿作成フォーム、POST /posts の投稿登録、/posts/{id} の投稿詳細を TDD で実装してください。Controller / Service / Repository のテストを追加し、./mvnw -B -Ph2 verify が通る状態にしてください。
```

**結果**: 効いた

投稿一覧、投稿作成フォーム、投稿登録、投稿詳細の基本導線を実装できた。
Controller / Service / Repository の各層にテストを追加し、画面遷移とビュー表示を確認できた。

**振り返り**:

受入基準と対象 URL を明示したことで、実装範囲がぶれにくかった。
次回は入力バリデーションの条件も最初から明記すると、手戻りをさらに減らせる。

---

## プロンプト 2

**フェーズ**: 投稿拡張機能

**プロンプト本文**:

```text
投稿詳細でいいねをトグルできるようにし、投稿一覧では本文キーワード検索を実装してください。削除は物理削除ではなく deleted_at による論理削除にしてください。Repository / Service / Controller のテストを追加して確認してください。
```

**結果**: 効いた

いいねの永続化、いいねトグル、本文キーワード検索、投稿の論理削除を追加できた。
削除済み投稿を一覧・検索・詳細から除外する動きもテストで確認できた。

**振り返り**:

複数機能を 1 つの依頼にまとめたため、コミット数は増えたがレビュー観点は少し広くなった。
次回は「いいね」「検索」「論理削除」を分けて依頼すると、RED / GREEN / REFACTOR の粒度がより明確になる。

---

## プロンプト 3

**フェーズ**: 表示設定・UI 改善

**プロンプト本文**:

```text
アプリ全体に適用される利用者ごとの表示設定として背景カラーを実装してください。/posts の右上で なし / 青 / 緑 / ピンク / グレーを選択でき、/posts/new と /posts/{id} にも反映されるようにしてください。不正値は なし に戻し、投稿データには保存しないでください。
```

**結果**: 部分的に効いた

最初は Cookie と CSS クラスでテーマカラーを保持・反映する実装になった。
その後、画面に反映されないケースと選択肢が英語表示になっている点を見直し、HttpSession による保持と body の background-color 出力に変更した。
選択肢も投稿者名の色選択に合わせて、青 / 緑 / ピンク / グレーの日本語表示に修正できた。

**振り返り**:

「設定直後に画面へ反映されること」と「表示名は日本語にすること」を最初の受入基準へ入れるべきだった。
UI の表示文言と状態保持方式は、実装前に具体例まで指定すると意図が伝わりやすい。
